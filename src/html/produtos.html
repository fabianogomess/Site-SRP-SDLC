PRODUCTS

  • SoPaMM (Software Pattern MetaModel)

    SoPaMM represents our attempt to extend SRP's benefits to other SDLC stages besides RE. It defines how functional requirement patterns (FRP) and acceptance test patterns (ATP) can be written, organized, related, and classified. It also borrows concepts and practices from Behavior-Driven Development (BDD) by describing FRP via user stories and associating it with behaviours through the Gerkhin language.

  • TMEd (Terminal Model Editor)

    TMEd is a software tool developed to produce pattern catalogues following the SoPaMM metamodel grammar. We hope the triad of the SoPaMM metamodel, SRP catalogues, and the TMEd tool help better understand the impact of SRP usage in the software lifecycle beyond RE.

  • MQuaRE (Metamodel Quality evaluation and REquirements)

    MQuaRE is a metamodel evaluation framework composed of metamodel quality requirements, a metamodel quality model, metamodel quality measures, and an evaluation process, with a great contribution of the ISO/IEC 25000 series for software product quality evaluation. MQuaRE can help determine final metamodel quality, decide on the acceptance of a metamodel, and also assess the positive and negative aspects of a metamodel, contributing to its quality evolution.

  • DREAM (behavior-DRivEn Application Model generator)

    DREAM is a tool developed to automate test cases and requirements specifications generation through SRP catalogues built using TMEd. It provides a set of functionalities such as basic user, project, and SRP catalogue management, translation from catalogue elements to software requirements and test cases, among others.
